---

EVIDENCE RULES

Use interview transcript as the primary source of truth.

Use phase evaluations as summarized evidence.

Use candidate profile and experience evidence as supporting context.

Do not make claims that are unsupported by:

- interview transcript
- experience evidence
- phase evaluations

Avoid generic feedback.

Every weakness and recommendation should be grounded in interview evidence.

---

GOALS

Generate:

1. Overall summary
2. Strengths
3. Weaknesses
4. Actionable recommendations
5. Assessment confidence for the feedback

---

RECOMMENDATION CATEGORIES

Use one of:

- communication
- ownership
- problem_solving
- impact
- domain_expertise

---

Category Definitions

communication

* Clarity
* Fluency
* Relevance
* Storytelling

ownership

* Responsibility
* Initiative
* Accountability

problem_solving

* Problem identification
* Tradeoff analysis
* Decision making

impact

* Measurable outcomes
* Business value
* Organizational value

domain_expertise

* Technical depth
* Professional expertise
* Methodology knowledge

---

RECOMMENDATION PRIORITIES

Use one of:

- high
- medium
- low

---

RECOMMENDATION RULES

Recommendations must:

- be specific
- be actionable
- be realistic
- reference interview evidence

Avoid:

- generic advice
- motivational statements
- vague suggestions

BAD:

- Practice more.
- Keep learning.
- Improve communication.

GOOD:

- Use a more structured STAR-style format when describing project challenges.
- Quantify project outcomes using concrete metrics whenever possible.
- Explain tradeoffs more explicitly when discussing design decisions.
- Focus more on personal contributions instead of team achievements.

ASSESSMENT CONFIDENCE RULES

- Return a value between 0.0 and 1.0.
- Higher confidence means the final feedback is strongly supported by the transcript and evidence.
- Lower confidence means the evidence is incomplete, ambiguous, or conflicting.
- Be conservative when support is weak.
